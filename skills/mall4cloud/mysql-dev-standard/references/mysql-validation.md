# MySQL 设计与变更检查清单

按变更范围执行，不适用项标记原因。高风险操作必须由 DBA 在目标环境执行或批准。

## 1. 范围与契约

- [ ] 已确认独立仓库、Maven 模块、数据库、表和业务负责人。
- [ ] 已列出 model、DTO/VO、Mapper/XML、Service、API 和调用方影响。
- [ ] 已确认 MySQL 版本、代理/分片环境、字符集和时区。
- [ ] 已估算当前行数、增长速度、读写 QPS、峰值和数据保留期。

## 2. 命名与表基线

- [ ] 标识符为小写 snake_case、含义清晰且不超过 32 字符。
- [ ] 未使用保留字、`test` 前缀或不合规临时对象命名。
- [ ] 引擎为 InnoDB，字符集为 `utf8mb4`，collation 与目标库一致。
- [ ] 表和非主键字段有准确 comment。
- [ ] 表有主键且未新增数据库外键。
- [ ] 新业务表包含五个基础字段，并与公共模型类型一致。
- [ ] 单库表数、单表字段数和预计行数未超过规范阈值。

## 3. 字段

- [ ] 每个字段使用满足业务范围且与调用方兼容的数据类型。
- [ ] 金额和精确数值使用 DECIMAL，并明确精度与小数位。
- [ ] VARCHAR 长度来自真实业务上限，不使用无依据的大长度。
- [ ] NOT NULL、默认值和空值语义不会混淆业务状态。
- [ ] 时间字段的时区、范围、精度和序列化方式已确认。
- [ ] TEXT/BLOB、超长字段、ENUM 或哈希伪列已取得必要评审。
- [ ] 文件或图片未直接保存到数据库。

## 4. 索引

- [ ] 索引总数不超过 5，联合索引字段不超过 5；例外有评审记录。
- [ ] 索引名称符合 `idx_` 或 `uk_` 约定。
- [ ] 没有重复、冗余或无查询依据的索引。
- [ ] JOIN 字段类型完全一致且访问路径可用。
- [ ] 联合索引顺序与等值、范围、排序和最左前缀匹配。
- [ ] 已评估区分度、覆盖率、回表、写放大和存储成本。
- [ ] 替代旧索引时先验证新索引，再安排安全下线旧索引。

## 5. SQL 安全与正确性

- [ ] 所有值参数均使用 prepared statement 或安全参数绑定。
- [ ] 动态表名、列名和排序字段只来自服务端白名单。
- [ ] INSERT 明确列名，未使用全字段复制。
- [ ] UPDATE/DELETE 有有界、可命中索引的 WHERE 条件。
- [ ] 已设置执行前影响行数核对和执行后数据校验。
- [ ] 未出现空 WHERE、恒真条件、危险全表清空或多表 DML。
- [ ] SQL 异常会触发项目事务回滚，不被捕获后继续执行。

## 6. SQL 性能

- [ ] IN 参数、批次、分页行数和结果集大小在规范阈值内。
- [ ] 未使用 3 表及以上 JOIN、无边界子查询或超大事务。
- [ ] 索引列未被函数、运算或隐式类型转换包裹。
- [ ] 未使用前导通配符或未评估的负向条件。
- [ ] 深分页、实时总量统计和大批量操作采用已评审方案。
- [ ] EXPLAIN 已记录 access type、possible_keys、key、rows 和 Extra。
- [ ] 全表扫描、filesort、temporary 或高 rows 估算均有处理或评审结论。

## 7. DDL/DML 变更

- [ ] 脚本不携带库名前缀、反引号或不必要特殊字符。
- [ ] 新增字段未使用 `AFTER`，或已确认代理兼容。
- [ ] 已评估元数据锁、表重建、磁盘空间、主从延迟和执行窗口。
- [ ] 已准备前向脚本、回滚脚本或明确补偿方案。
- [ ] 工单包含申请人、目标环境、完整脚本、计划时间、影响和耗时。
- [ ] 高风险脚本由 DBA 执行或批准，未在业务代码中自动运行。

## 8. 验证与汇报

- [ ] 已运行目标模块最小单元测试和 Mapper/XML 契约测试。
- [ ] 已验证新增字段的读写、默认值、历史数据和序列化兼容。
- [ ] 已验证成功、失败、回滚、并发和边界数据。
- [ ] 汇报准确命令、结果、未执行项、DBA 确认项和上线观察指标。
