# Redis Dev Standard

## 适用范围

用于工作区内所有 Redis 设计与变更，尤其是 Mall4Cloud 后端缓存、计数、分布式状态、限流、Key 迁移、Lua 脚本和故障降级。

规范源 `REDIS-STD-V2026070301` 当前标记为草稿。实现时同时遵循目标仓库现有缓存封装、Redis 实际版本/拓扑和 DBA 已确认要求；Redis 不得成为强事务或强持久数据的唯一事实来源。

## 工作顺序

1. 确认数据事实来源、业务所有者、实例、环境、预估 QPS、容量和一致性要求。
2. 选择数据结构，设计 Key 命名、Value 大小、TTL、失效/回填和租户隔离。
3. 检查命令复杂度、批次、响应大小、集群 slot、Lua、并发重建和故障路径。
4. 评估缓存穿透、击穿、雪崩、热点、BigKey、连接池和重试放大风险。
5. 对生产扫描、迁移、淘汰策略、危险命令管理和 BigKey 清理取得 DBA/运维确认。

## 必须遵循

- Redis 默认用于高频、低延迟、可重建数据的缓存，不承载必须强事务、强持久的唯一数据副本。
- 无关业务不得共用同一 Redis 实例；Key 必须包含清晰业务命名空间，并检查环境、租户和资源归属隔离。
- Key 使用冒号分隔的短而可读结构，禁止空格、换行、单双引号和转义字符。
- String 序列化后控制在 10 KB 内；Hash/List/Set/ZSet 单结构元素不超过 5000。
- 禁止直接 `DEL` 超大聚合结构；使用经 DBA 审核的渐进清理，版本支持且适用时优先异步删除。
- 缓存 Key 必须设置与业务一致的 TTL；批量 Key 的过期时间加入抖动，避免集中失效。
- 超过 14 天或永久 Key 必须取得 DBA 确认，并持续监控内存和闲置时长。
- 线上禁止直接执行 `KEYS`、`FLUSHALL`、`FLUSHDB`；全量检索使用有界 `SCAN` 系列并由运维控制。
- HGETALL、HKEYS、HVALS、LRANGE、SMEMBERS、ZRANGE、SINTER 等 O(N) 操作必须有明确数据量上限；大 Hash/Set/ZSet 使用 HSCAN/SSCAN/ZSCAN 分段处理，大 List 使用经审核的有界读取、LTRIM 分段清理或适用版本下的异步删除方案。
- 批量操作采用原生批量命令或 Pipeline；按来源中更严格阈值，单批不超过 100 个元素且总数据量不超过 10 KB。
- 集群 Lua 的所有 Key 必须通过 `KEYS` 数组传入，脚本访问的 Key 必须落在同一 slot。
- 客户端必须使用项目管理的连接池/客户端封装，配置超时，捕获操作异常并在 `finally` 或框架生命周期中归还资源。
- 日志只记录必要的 Key 标识和错误上下文，不记录密码、完整 Value、用户隐私或敏感凭据。

## 数据结构选择

- **String**：简单 KV、计数和短文本。优先存数字；长文本先评估压缩和是否应进入其他存储。
- **Hash**：实体对象或关联属性，避免把同一对象拆成大量独立 String。
- **Set**：去重、标签和无序成员关系。
- **Sorted Set**：排行榜、权重和有序成员关系。
- **List**：有序列表；不得作为关键可靠消息队列。临时、可丢失场景也需确认生产消费速率平衡。
- **特殊结构**：HyperLogLog、Geo、Lua/Scripting 等使用前与 DBA/架构确认精度、slot 和运维影响。

## 生命周期与一致性

- 明确缓存命中、未命中、过期、主动失效、更新失败和 Redis 不可用时的行为。
- 缓存更新必须与数据库事实来源建立可解释的一致性策略，避免写成功但缓存状态不确定。
- 核心链路提供超时、熔断、降级或回源保护；禁止无界重试拖垮 Redis 和数据库。
- 并发回填使用项目既有互斥、逻辑过期或单飞机制，避免缓存击穿。
- 对空结果、非法 ID 和恶意请求采取有界负缓存或前置校验，避免缓存穿透。

## 命令与集群

- 原生 MGET/MSET 等批量命令具有对应命令语义；Pipeline 仅减少网络往返，不保证整批原子性。
- 少用 Redis 事务；事务不支持传统数据库式回滚，集群事务所有 Key 必须同 slot。
- `MONITOR` 只能由运维短时采样，不得在业务流程或长期诊断中运行。
- 谨慎使用 SELECT。业务隔离依赖独立实例/集群和命名空间，不依赖数字数据库。

## 客户端与运维边界

- 高并发关键链路必须有项目认可的熔断降级，不机械引入来源中的旧组件示例。
- 实例认证、传输加密、网络访问控制和凭据管理由平台规范配置，代码和 Skill 不保存真实凭据。
- `maxmemory-policy` 根据数据是否均有 TTL、事实来源和降级行为由 DBA 选择，不假设某个策略是所有环境默认值。
- 生产迁移、危险命令屏蔽、热点分析、实例扫描和 BigKey 删除属于 DBA/运维操作。

## 来源例外勿复制

- 来源同时给出 String 1 MB 与 10 KB、集合 10000 与 5000、批次 100 与 500 等阈值；新设计按 10 KB、5000、100 的更严格值检查，例外由 DBA 确认。
- 来源一处禁止 Redis 队列，另一处将 List 用作临时队列；本 Skill 禁止关键可靠队列，只允许经过评审的非关键临时场景。
- 不复制 `HMSET`、旧熔断组件、明文连接参数或直接创建 Jedis 的示例；使用项目当前客户端 API。
- 不复制逐元素同步删除的示例代码到生产业务；BigKey 清理由运维运行已验证、可暂停、可观测的工具或脚本。
