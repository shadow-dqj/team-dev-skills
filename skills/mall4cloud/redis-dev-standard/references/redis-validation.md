# Redis 设计与上线检查清单

按变更范围执行，不适用项标记原因。生产扫描、迁移、配置和 BigKey 清理必须由 DBA/运维批准。

## 1. 选型与范围

- [ ] 已确认数据事实来源，Redis 不承载强事务或强持久的唯一副本。
- [ ] 已确认业务、环境、实例、租户、数据等级和责任人。
- [ ] 已估算峰值 QPS、Key 数、内存、Value 大小、元素数和增长速度。
- [ ] 已确认 Redis 版本、Cluster/主从拓扑、客户端和托管平台限制。
- [ ] 关键可靠队列未使用 Redis；临时 List 场景有明确可丢失语义。

## 2. Key 与数据隔离

- [ ] Key 使用短而清晰的冒号命名空间。
- [ ] Key 不含空白、引号、转义字符或敏感明文。
- [ ] 多环境、多业务和多租户隔离不可由调用方绕过。
- [ ] 多 Key 操作的 hashtag 与 slot 分布已验证。
- [ ] 无关业务未共享同一故障域或实例。

## 3. Value 与数据结构

- [ ] 数据结构与访问模式匹配，没有把对象拆成大量无关 String。
- [ ] String 序列化后不超过 10 KB。
- [ ] Hash/List/Set/ZSet 元素不超过 5000。
- [ ] 已检查压缩的 CPU 成本、序列化兼容和最大返回体。
- [ ] 特殊结构已取得 DBA/架构确认。
- [ ] 不存在需要直接同步 DEL 的 BigKey。

## 4. TTL 与一致性

- [ ] 所有缓存 Key 有与业务一致的 TTL。
- [ ] 超过 14 天或永久 Key 有 DBA 确认、监控和清理责任人。
- [ ] 批量 Key 的 TTL 有随机抖动。
- [ ] 已定义命中、未命中、过期、主动失效和回填流程。
- [ ] 已定义数据库提交与缓存更新/失效顺序。
- [ ] 更新缓存失败有补偿、重试上限或可接受的一致性说明。
- [ ] 并发重建有互斥、单飞或逻辑过期保护。

## 5. 命令与批量

- [ ] 批量命令不超过 100 个元素且总数据量不超过 10 KB。
- [ ] O(N) 命令有明确成员数、返回数、字节数和耗时上限。
- [ ] 大 Hash/Set/ZSet 使用对应 SCAN 命令，大 List 使用有界读取或分段 LTRIM，不使用 KEYS。
- [ ] 业务流程不调用 FLUSHALL、FLUSHDB 或长期 MONITOR。
- [ ] Pipeline 调用方逐项处理结果，不假设整批原子性。
- [ ] 事务使用理由充分，Cluster Key 位于同一 slot。
- [ ] Lua 的全部 Key 从 KEYS 传入且位于同一 slot。
- [ ] Lua 有执行时间、循环次数、返回大小和失败处理边界。

## 6. 客户端与故障

- [ ] 使用项目管理的连接池/客户端封装。
- [ ] 连接、命令、池等待超时和最大连接数已配置。
- [ ] 重试有次数、退避和总耗时上限，不会放大流量。
- [ ] 所有异常路径正确归还资源，不吞掉关键失败。
- [ ] 关键链路有熔断、限流、降级或受保护的数据库回源。
- [ ] Redis 完全不可用时，业务行为和容量保护已验证。
- [ ] 日志和指标不包含凭据、完整 Value 或用户隐私。

## 7. 缓存风险

- [ ] 穿透防护覆盖非法参数和持续不存在的数据。
- [ ] 击穿防护覆盖热点 Key 过期和并发回填。
- [ ] 雪崩防护覆盖集中失效、预热失败和实例故障。
- [ ] 热点 Key 有识别指标和拆分/合并/降级方案。
- [ ] BigKey 有大小、元素数和延迟监控。
- [ ] 淘汰策略来自目标实例真实配置并符合业务语义。

## 8. 生产运维

- [ ] 生产扫描使用限速、有界、可停止的运维流程。
- [ ] BigKey 清理按类型渐进执行，或使用经确认的异步删除。
- [ ] 清理流程能处理并发写入、部分失败和重复执行。
- [ ] 迁移、危险命令管理、TLS 和淘汰策略变更由 DBA/运维实施。
- [ ] 已准备回滚/停止条件、观察窗口和责任人。

## 9. 测试与监控

- [ ] 已测试命中、未命中、过期、空值、脏值和序列化失败。
- [ ] 已测试超时、连接拒绝、实例不可用和 Cluster slot 错误。
- [ ] 已测试并发回填、数据库压力保护和降级恢复。
- [ ] 已观察命中率、内存、eviction、连接池、命令延迟和错误率。
- [ ] 汇报准确命令、结果、未执行项、DBA 确认项和人工验证要求。
